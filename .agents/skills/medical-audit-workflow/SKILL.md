# 医保审核工作流

执行完整的医保审核流程，包含三个核心步骤。每个步骤执行后记录详细日志，包括执行状态、时间戳、结果文件路径和耗时。

## 执行流程

### 步骤1: 同步审核数据到 DuckDB

调用 MCP 工具 `sync_audited_result_data`，将审核数据导出为 Parquet 文件并注册为可查询视图。

**执行逻辑**:
1. 调用 `mcp_Guo_Xin_Shen_He_Shu_Ju_Fu_Wu` 服务的 `sync_audited_result_data` 工具
2. 等待执行完成
3. 记录执行状态（成功/失败）
4. 记录返回的文件路径和视图名称
5. 记录执行时间戳和耗时

**日志格式**:
```
[步骤 1/3] 同步审核数据到 DuckDB
- 状态: 成功/失败
- 执行时间: YYYY-MM-DD HH:MM:SS
- 结果: [文件路径/视图名称/错误信息]
- 耗时: X 秒
```

### 步骤2: 执行审核分析

调用 MCP 工具 `execute_duck_db_query_by_dimension`，获取所有可用的分析维度。

**执行逻辑**:
1. 调用 `mcp_Guo_Xin_Shen_He_Shu_Ju_Fu_Wu` 服务的 `execute_duck_db_query_by_dimension` 工具
2. 等待执行完成


**参数说明**:
- `Name`: 分析维度名称
- `Description`: 分析维度描述
- `DataType`: 数据类型

**日志格式**:
```
[步骤 2/3] 获取可用分析维度
```

### 步骤3: 获取步骤需要分析维度的结果

调用 MCP 工具 `execute_duck_db_query_by_dimension`，将审核结果导出为 Excel 文件。

**执行逻辑**:
1. 检查用户是否提供了过滤条件参数
2. 根据步骤2获取的分析维度名称对应入参

**参数说明**:
- `dimensionName`: 分析维度名称（唯一标识），从 get_available_analysis_dimensions 返回的 Name 字段获取
"）

**日志格式**:
```
[步骤 3/3] 导出 Excel 结果
-返回分析维度结果的JSON数据结构
```

## 错误处理

### 步骤失败处理
- 如果任一步骤失败，记录详细错误信息
- 停止执行后续步骤
- 输出失败步骤编号和错误详情
- 提供可能的解决方案建议

### 常见错误场景
1. **MCP 服务不可用**: 检查 MCP 服务是否已启动并正确配置
2. **数据库连接失败**: 检查 DuckDB 数据库连接是否正常
3. **参数无效**: 验证提供的规则编码、医院编码等参数是否有效
4. **文件路径无效**: 检查临时文件夹路径是否可访问

## 成功完成

所有步骤成功后，输出汇总信息：

```
医保审核流程执行完成
================================
总耗时: X 秒
生成文件:
1. DuckDB 视图: [视图名称]
2. 分析维度名称: [分析维度名称]
3. 根据分析维度名称得到结果的JSON数据结构
================================
```

## 使用示例

### 基本触发
- "执行医保审核流程"
- "运行审核数据同步"
- "触发医保审核"

### 带参数触发
- "执行医保审核，规则编码：RULE001"
- "同步审核数据并导出到Excel，医院编码：H001"
- "执行医保审核，规则编码：RULE001，医院编码：H001，工作表名称：审核结果"

### 带过滤条件触发
- "导出审核结果，病案号：MR001"
- "导出审核结果，参保人号：INS001，规则编码：RULE001"

## 注意事项

1. **执行顺序**: 必须严格按照步骤1 → 步骤2 → 步骤3的顺序执行
2. **依赖关系**: 步骤2和步骤3依赖步骤1的数据同步结果
3. **参数传递**: 用户提供的参数应在相应步骤中正确传递
4. **日志记录**: 每个步骤执行后必须立即输出日志，不要等到所有步骤完成
5. **文件路径**: 所有生成的文件路径由 MCP 服务自动管理，记录返回的路径即可
